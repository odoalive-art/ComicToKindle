# ComicToKindle 路线图

本文是 ComicToKindle 的路线图源文件。未来上线官网后可渲染为 `/roadmap`。

路线图用于表达方向，不承诺具体日期。状态含义：

- **已上线**：已经进入可用内测包的能力
- **正在打磨**：功能已存在或主路径已打通，但仍在稳定性、体验和边界处理上持续改进
- **计划中**：产品方向明确，尚未进入完整实现
- **探索中**：价值明确但方案未定，可能被拆分、延期或调整

## 已上线

### 本地漫画库

- 左侧文件夹树 + 忠实磁盘目录网格；递归识别本地漫画目录用于可读卷册、封面和转换元信息。
- 左侧文件夹树导航：忠实磁盘定位，点文件夹后内容区复用统一网格展示直接文件/文件夹，支持选中、右键和拖拽到树节点/文件夹整理。
- 支持封面缩略图缓存。
- 支持部名称和作者在应用内编辑，不修改本地文件夹名。
- 支持应用内文件整理：重命名、移动（含拖拽移动）、新建文件夹、删除到废纸篓。

### 阅读与来源

- 支持卷册阅读器：单页/双页、左右阅读方向、续读进度。
- 支持图片目录卷册。
- 支持 CBZ/ZIP/CBR/RAR/7z 压缩包卷册。
- 支持加密 zip、多卷分卷、共享密码池和解压进度。
- 支持 PDF 单文件卷册，渲染为页面图片后阅读与转换。
- 支持图片型 EPUB 来源，按 OPF spine / XHTML 图片引用抽取页面。

### Kindle 转换

- 支持图片卷册转换为 Kindle 固定版式 EPUB。
- 支持转换前确认书籍信息。
- 支持写入「漫画名 + 卷册」书名和作者元数据。
- 支持转换队列持久化、中断提示和用户确认继续。

### 归档与投递

- 支持转换产物归档管理。
- 支持在 Finder 显示、导出副本、删除产物。
- 支持 SMTP 邮件投递到 Kindle。
- 支持 Send to Kindle 网页通道，适合较大的 EPUB 产物。

### 内测打包

- macOS 内测包走 ad-hoc 签名 dmg。
- dmg 文件名包含版本号和构建时间戳，便于留存对比。
- dmg 内置当前版本说明。
- 打包前可运行 `npm run pack:doctor` 做环境和配置体检。

## 正在打磨

### 转换稳定性

- 压缩包兼容性和异常提示。
- PDF 渲染耗时、字体/特殊页面兼容性和失败提示。
- 图片型 EPUB 的目录结构兼容性；纯文本/重排 EPUB 保持明确失败说明。
- 大卷册转换耗时与进度反馈。
- 转换失败后的错误说明和恢复路径。
- 产物命名、归档状态和重复转换体验。

### 阅读体验

- 阅读器翻页、方向、双页模式细节。
- 大图加载体验和空白/损坏图片反馈。
- 续读状态的边界情况。

### 文件管理体验

- 多选、框选、右键菜单和移动弹框细节。
- 文件操作后的刷新一致性。
- 删除、重命名、移动失败时的用户提示。

### 内测发布流程

- 更新日志和路线图维护规范。
- dmg 内容、版本号、构建标识和包体大小检查。
- 未来官网 `/changelog` 和 `/roadmap` 的内容同步。

## 计划中

### 元数据与索引

- 建立本地元数据存储，减少每次进入库视图的全量实时扫描成本。
- 支持漫画、卷册、作者、转换状态和阅读状态的稳定索引。
- 为后续搜索、筛选、排序和批量管理打基础。

### 转换后自动投递

- 转换完成后按用户设置自动进入 SMTP 或网页推送流程。
- 支持按单卷或批量任务选择自动投递策略。
- 保留手动确认和失败重试路径。

### 导入与整理

- 支持外部拖拽导入卷册或文件夹。
- 支持导入时选择目标部或新建部。
- 支持导入后自动刷新库视图。

## 探索中

### 图像增强

- 图像放大或降噪流程。
- 面向 Kindle 屏幕的清晰度优化。
- 处理速度、包体和本地资源占用的平衡。

### Kindle 预览

- 转换前或转换后模拟 Kindle 阅读效果。
- 页面尺寸、双页方向、封面和书名显示预览。
- 用于减少反复转换试错。

### 多格式预览

- 对压缩包、PDF、EPUB 等不同来源提供统一的预览入口。
- 在不完整解析的情况下给出可理解的格式状态。

### 更完整的发布站点

- 官网首页、下载页、更新日志和路线图。
- 内测下载说明、Gatekeeper 打开说明和常见问题。
- 后续正式分发、签名、公证和自动更新策略。
