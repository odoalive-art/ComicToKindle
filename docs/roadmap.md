# ComicToKindle 路线图

本文是 ComicToKindle 的路线图源文件。未来上线官网后可渲染为 `/roadmap`。

路线图用于表达方向，不承诺具体日期。状态含义：

- **已上线**：已经进入可用内测包的能力
- **正在打磨**：功能已存在或主路径已打通，但仍在稳定性、体验和边界处理上持续改进
- **计划中**：产品方向明确，尚未进入完整实现
- **探索中**：价值明确但方案未定，可能被拆分、延期或调整

## 已上线

### 本地漫画库

- Eagle 式 `.ctklib` 托管库包浏览：导入即复制成桶，manifest 投影为「文件夹 + 单册书」书架，支持多级下钻、面包屑、选中、右键整理和排序。模型为「书（单册，自带书名+作者）+ 文件夹（纯收纳，无作者，作者跟书走）」。
- 支持每本书书名+作者编辑（单本「编辑信息」/ 多选「批量设置作者」）、文件夹重命名、分组归属在应用内编辑；整理只写托管库 manifest 或移动库内桶，不修改用户导入源。
- 支持导入预览时选择目标：散卷、已有文件夹或新建文件夹；支持从 Finder/桌面把文件或文件夹拖放到库内容区导入。
- 支持封面缩略图缓存。
- 支持库内回收站：书软删除到 `.ctklib/trash/`，可还原或清空。

### 阅读与来源

- 支持卷册阅读器：单页/双页、左右阅读方向、续读进度。
- 支持图片目录卷册。
- 支持 CBZ/ZIP/CBR/RAR/7z 压缩包卷册。
- 支持加密 zip、多卷分卷、共享密码池和解压进度。
- 支持 PDF 单文件卷册，渲染为页面图片后阅读与转换。
- 支持图片型 EPUB 来源，按 OPF spine / XHTML 图片引用抽取页面。

### 阅读时 AI 放大（高清化）

- 阅读漫画时可一键开启 AI 放大（waifu2x-ncnn-vulkan，本地离线、GPU 加速），挂在统一取图协议上，对所有来源格式生效。
- 按需增强 + 内容寻址磁盘缓存 + 相邻页预取，配合「原图秒显 → 增强替换」做到准实时。
- 阅读器内角标显示当前页增强状态（已增强 / 增强中 / 失败），并支持「按住对比原图」直接核对效果。
- 扩展功能页可配置倍率、降噪、缓存上限并一键清理；自动探测 GPU/CPU 并在仅 CPU 时提示降级。

### Kindle 转换

- 支持图片书转换为 Kindle 固定版式 EPUB。
- 转换走「格式转换」独立模态弹窗，逐本预填书名/作者。
- 支持写入书名（单一字段）和作者元数据。
- 支持转换队列持久化、中断提示和用户确认继续。

### 归档与投递

- 支持转换产物归档管理。
- 支持在 Finder 显示、删除产物。
- 支持 SMTP 邮件投递到 Kindle。
- 支持 Send to Kindle 网页通道，适合较大的 EPUB 产物。

### 内测打包

- macOS 发布包可用稳定自签证书签名，并保留不传 `CTK_SIGN_IDENTITY` 时的 ad-hoc 本地内测路径。
- GitHub Release 同时发布 dmg、Squirrel.Mac zip 与 `latest-mac.yml`；electron-updater 已完成 `0.1.0-beta.3 → 0.1.0-beta.4` 真实升级验证。
- dmg 文件名包含版本号和构建时间戳，便于留存对比。
- dmg 内置当前版本说明。
- 打包前可运行 `npm run pack:doctor` 做环境和配置体检。

## 正在打磨

### 转换稳定性

- 压缩包兼容性和异常提示。
- PDF 渲染耗时、字体/特殊页面兼容性和失败提示。
- 图片型 EPUB 的目录结构兼容性；纯文本/重排 EPUB 保持明确失败说明。
- 大卷册转换耗时与进度反馈。
- 转换失败后的错误说明和恢复路径。
- 产物命名、归档状态和重复转换体验。

### 阅读体验

- 阅读器翻页、方向、双页模式细节。
- 大图加载体验和空白/损坏图片反馈。
- 续读状态的边界情况。

### 库整理体验

- 多选、框选、右键菜单和移动弹框细节（含 Shift 范围多选、Cmd/Ctrl+A 全选漫画、Delete 删除快捷键）。
- 桌面化交互收敛：禁用网页式文字选择与浏览器快捷键，Cmd/Ctrl+R 复用为重命名。
- 删除部可选「仅解散」或「连内部卷册一并移入库内回收站」。
- manifest 整理后的刷新一致性。
- 删除、重命名、归属调整失败时的用户提示。

### 内测发布流程

- 更新日志和路线图维护规范。
- dmg 内容、版本号、构建标识和包体大小检查。
- 继续观察公开 GitHub Release 的稳定性，并在后续版本发布时复验匿名 feed 与自签身份。
- 未来官网 `/changelog` 和 `/roadmap` 的内容同步。

## 计划中

### 元数据与索引

- 建立本地元数据存储，减少仅靠 `.ctklib` manifest 与桶 `book.json` 投影时的扩展成本。
- 支持漫画、卷册、作者、转换状态和阅读状态的稳定索引。
- 为后续搜索、筛选、排序和批量管理打基础。

### 转换后自动投递

- 转换完成后按用户设置自动进入 SMTP 或网页推送流程。
- 支持按单卷或批量任务选择自动投递策略。
- 保留手动确认和失败重试路径。

## 探索中

### 转换时图像增强

- 把 AI 放大烤进转换出的 Kindle 固定版式 EPUB，让 Kindle 上直接拿到 HD 版（区别于已上线的「阅读时增强」，后者只影响应用内观看）。
- 更多放大模型与降噪策略；处理速度、包体和本地资源占用的平衡。

### Kindle 预览

- 转换前或转换后模拟 Kindle 阅读效果。
- 页面尺寸、双页方向、封面和书名显示预览。
- 用于减少反复转换试错。

### 多格式预览

- 对压缩包、PDF、EPUB 等不同来源提供统一的预览入口。
- 在不完整解析的情况下给出可理解的格式状态。

### 更完整的发布站点

- 官网首页、下载页、更新日志和路线图。
- 内测下载说明、Gatekeeper 打开说明和常见问题。
- 后续正式分发、签名、公证和自动更新策略。

### 应用内访问 KMOE + 自动导入

- 在应用内嵌入 KMOE（kox.moe）漫画站,登录后直接浏览/下载（KMOE 下载即为 EPUB,源图命名 `moe-*.jpg`,与现有 EPUB 导入管线天然契合）。
- 默认把下载到本地的书籍自动导入应用库（候选实现:拦截 Electron `session` 的 `will-download` 直接落到导入目录并触发 `library:import`,或监听下载目录）。
- 待定:登录态/会话持久化、嵌入方式（`WebContentsView`/`<webview>`）、自动导入的目标分组与去重策略。
- 优先级:已登记需求,暂不排期。
